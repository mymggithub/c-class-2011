How to Dynamically Allocate an Array Using Class in C++

In basic C++ programming, variables and arrays are declared by the programmer directly in code. However, when building data structures such as dynamically resizing arrays, the programmer must learn to dynamically allocate memory during code execution. Arrays allocated in this fashion have a few benefits, such as non-constant length at declaration. The programmer can allocate arrays of scalar data types and user-created classes identically. Using pointers and keywords built into the standard library, C++ makes memory allocation and deletion straightforward.

  • Text Editor

Instructions

    • 1

      Create a pointer of the data type the array will contain. For example, to dynamically create an array of integers, an integer pointer must exist to reference the array:

      int * arr_ptr;

      The pointer "arr_ptr" will now allow a programmer to reference deep memory during execution.

    • 2

      Use the pointer to allocate memory. The "new" keyword, which is specific to the C++ libraries (and not available in standard C) claims memory during execution for use of the calling pointer:

      arr_ptr = new int [6];

      Here, the "new" keyword grabs a block of six integer-sized sections of memory. However, unlike arrays declared before run time, dynamic arrays can be declared with variable input:

      int i = 6;

      arr_ptr = new int [i];

      This allows the programmer to create arrays that can resize based on need rather than predetermined space designation.

    • 3

      Allocate an array of user-created data types. Following C++ memory allocation procedure, the programmer can create arrays of data types declared in separate classes:

      int i = 5;

      newClass * class_ptr;

      class_ptr = new newClass [i];

      The pointer "class_ptr" will reference a variable of the "newClass" data type, most likely defined in a class description in another file.

    • 4

      Delete the array. When an array is no longer needed (or has been copied into another array, possibly during a resizing operation) the programmer should make sure to delete it. Allocated memory that is not deleted, but no longer referenced, can take up valuable memory and, over time, cause what is known as a memory leak.

      int i = 5;

      newClass * class_ptr;

      class_ptr = new newClass [i];

      delete class_ptr[];

      Now, the array referenced by "class_ptr" will be deleted, and the memory freed for other uses.
